La red social DelayedMeter maneja los perfiles de sus usuarios en una base de datos que contiene numRegistros registros continuos. Su gestor de base de datos está fallando, por lo tanto te piden migrar los datos provisionalmente a un programa en lenguaje C.

  • char nombreUsuario[]: El nombre del i-ésimo usuario (máximo 20 caracteres).
  • char correo[]: El correo electrónico del i-ésimo usuario (máximo 64 caracteres).
  • char contrasena[]: La contraseña del i-ésimo usuario (máximo 32 caracteres).
  • int publicaciones: La cantidad de publicaciones del i-ésimo usuario.

Una vez que implementaste el tipo de dato abstracto, para probar tu implementación, te dan una serie de consultas modeladas con el siguiente tipo de dato abstracto

struct Consulta{ char tipo[10]; char campo[20]; char valor[65]; }

  • tipo: Solo puede ser "SELECT" o "MODIFY"
    • "SELECT": Con este tipo de consulta, el valor de campo es nulo y valor es un índice entre 0 y numRegistros-1 siendo el índice del arreglo al que aplicarás las consultas de tipo"MODIFY"
    • "MODIFY": Con este tipo de consulta el valor de
    • campo es una cadena con el campo que se quiere modificar. Este tipo de consultas solo pueden aparecer previas a un "SELECT" y actúan sobre el último elemento seleccionado con el otro comando.
  • campo: Puede tener los siguientes campos
    • "username": Cambia el nombre de usuario
    • "email": Cambia el email
    • "password": Cambia la contraseña, adicionalmente, se debe validar si la contraseña no es la misma.
    • "new_post": Incrementa en 1 la cantidad de publicaciones, en este caso, valor es nulo.
  • valor: Contiene en valor a modificar para cualquiera de los campos ingresados

Entrada:

La función procesarConsultas() recibe los siguientes parámetros:

  1. int numRegistros: El número de registros en la base de datos.
  2. Perfil usuarios[]: El arreglo de estructuras con la información inicial de los usuarios.
  3. int numConsultas: La cantidad de operaciones a procesar.
  4. Consulta consultas[]: El arreglo de consultas a procesar.

Salida:

El programa debe generar reportes en la salida estándar (stdout) siguiendo estas reglas:

  1. Validación de Contraseñas: Durante el procesamiento de un MODIFY con campo "password", si la nueva contraseña (valor) es idéntica a la que el usuario ya tiene actualmente, se debe imprimir inmediatamente el siguiente error y NO realizar el cambio:

    Error: Usuario {username}, esa ya es tu contraseña

  2. Reporte Final: Una vez procesadas todas las consultas, debes imprimir la base de datos completa para verificar la migración. Cada usuario debe imprimirse en una línea con el siguiente formato:

    [{i}] {username} - {email} - {password} - {publicaciones} posts

    Donde {i} es el índice del usuario en el arreglo.

Se recomienda hacer uso de atoi y las funciones de la biblioteca string.h (no es necesario incluirla) para la realización de este ejercicio.

Consideraciones (no es necesario hacer validaciones)

  • int numRegistros - es un entero entre 1 y 500
  • int numConsultas - es un entero entre 1 y 200
  • "SELECT" siempre es el primer tipo de consulta
  • "MODIFY" actúa sobre el índice del último "SELECT"

Estás programando un módulo básico para gestionar tareas en un sistema operativo experimental llamado Doors. Tu compañero anterior te dejó implementados dos tipos de datos abstractos.

struct Proceso{

int pid;

char nombre[32];

};

struct GestorProcesos{

struct Proceso *lista; // Arreglo dinámico

int cantidad; // Ocupados actualmente

int capacidad; // Tamaño total reservado en memoria

};

Implementa las funciones para manejar estas dos estructuras, teniendo en cuenta que GestorProcesos contiene un arreglo dinámico.

Entrada

  • int numOperaciones - El número de operaciones que se harán sobre el gestor de procesos 
  • Operacion operaciones[] - La lista de operaciones que deberás ejecutar

 La estructura Operacion se define de la siguiente manera:

struct Operacion{

 char accion[10];  

 int parametro;

 char nombre[32];

}

Salida

Por cada operación de tipo "STATUS" muestra en la salida estándar los procesos activos con el siguiente formato:
"[{indice del proceso en el arreglo}] PID: {Proceso.pid} - {Proceso.nombre}"

Consideraciones (no es necesario hacer validaciones)

int pid es un entero entre 0 y 1000

char nombre[32]; es una cadena cualquiera de hasta 32 caracteres de longitud.

struct Proceso *lista; puede crecer hasta un tamaño de 100.

int cantidad; puede crecer hasta un tamaño de 100

int capacidad; puede crecer hasta 12800

int numOperaciones - es un entero entre 2 y 100

char accion[10] solo puede tener las siguientes operaciones: "INIT", "ADD", "KILL", "STATUS"

int parametro; Solo puede ser un proceso pid válido (existente actualmente), para las operaciones "KILL" y "ADD" o una capacidad entre 1 y 100 para la operación "INIT"

La operación "INIT" siempre es la primera operación.
